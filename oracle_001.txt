관계형 데이터베이스(RDB)와 NoSQL 의 개념 및 차이점을 공부한 후 정리
RDB(Relational DataBase)
: 모든 데이터를 테이블(표) 형태로 저장, 표현하는 관계형 데이터베이스

NoSQL(Not only SQL)
: JSON 객체와 유사한 문서 형태로 데이터를 저장하는 비관계형 데이터베이스 시스템

차이점
: 데이터를 저장하고 구성하는 방식
NoSQL은 JSON같은 문서 형태, RDB는 테이블 형태
NoSQL은 유연한 스키마 제공으로 변경이 쉽다.
RDB는 고정 스키마라 변경이 어렵다.
NoSQL은 BASE 원칙으로 데이터 중복의 가능성이 있고
RDB는 ACID를 보장해 데이터 무결성을 보장한다.
NoSQL은 데이터 구조가 미정이고, UPDATE가 적고, 조회가 많은 경우
RDB는 데이터 구조가 명확하고, UPDATE가 잦은 시스템에 좋다.
